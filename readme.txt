Описание FamilyTask (пока черновое)

1. Изначально открывается окно авторизации для входа или создания УЗ
2. После входа октроется профиль и кнопка для создания пары
3. При нажатии кнопки создания пары, открывается окно для поиска другой УЗ для создания пары
4. После создания пары появляется окно с таблицей задач (заголовки таблиц отображают текущий статус задачи (изначально три Backlog, В работе, Готово)) и кнопкой "Создать задачу"
5. Кнопка "Создать задачу" открывает форму со списком полей для создания задачи, и кнопками "ОК" и "Отмена"
6. При нажати кнопки "ОК" добавляется задача в таблицу задач в столбец Backlog
7. При клике мышкой на задачу появляется кнопка для перевода задачи на следующий статус
8. При переводе задачу на статус "В работе", исполнитель становится тот, кто нажал кнопку
9. При переводе задачи в статус "Готово", прописывается время завершения задачи
10. При создании задачи можно выбрать её сложность, приоритет, срок исполнения
11. Пара видеть только свои задачи, где участник пары является либо создателем, либо исполниетелем

Нужны следующие таблицы в БД

1. Пользователи (users)
2. Семья (family) - таблица отношений между пользователями
3. Список приоритетов (priority)
4. Список статусов для задач (status)
5. Задачи (tasks)
