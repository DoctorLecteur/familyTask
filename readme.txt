Описание FamilyTask (пока черновое)

1. Изначально открывается окно авторизации для входа или создания УЗ - готово
2. После входа октроется профиль и кнопка для создания пары - готово
3. При нажатии кнопки создания пары, открывается окно для поиска другой УЗ для создания пары - готово
4. После создания пары появляется вкладка с таблицей задач (заголовки таблиц отображают текущий статус задачи (изначально три Backlog, В работе, Готово)) и кнопкой "Создать задачу" - готово
5. Кнопка "Создать задачу" открывает форму со списком полей для создания задачи, и кнопками "ОК" и "Отмена" - готово
6. При нажати кнопки "ОК" добавляется задача в таблицу задач в столбец Backlog - готово
7. При клике мышкой на задачу появляется кнопка для перевода задачи на следующий статус - готово
8. При переводе задачу на статус "В работе", исполнитель становится тот, кто нажал кнопку - готово
9. При переводе задачи в статус "Готово", прописывается время завершения задачи - готово
10. При создании задачи можно выбрать её сложность, приоритет, срок исполнения - готово
11. Пара видет только свои задачи, где участник пары является либо создателем, либо исполниетелем - готово

Нужны следующие таблицы в БД

1. Пользователи (users)
2. Семья (family) - таблица отношений между пользователями
3. Список приоритетов (priority)
4. Список статусов для задач (status)
5. Задачи (tasks)
6. Список сложностей (complexity)
